KPI-MKT-001 دوره پایش: ماهانه دارای تغییرات ذخیره‌نشده

پیکربندی وب سرویس

فراخوانی همیشه با POST انجام می‌شود

Payload ارسالی همسو

فیلدها ثابت هستند و با مقدار شاخص جاری ارسال می‌شوند 
{
  "measureName": "نرخ رضایت مشتریان از خدمات پس از فروش",
  "measureCode": "KPI-MKT-001",
  "year": 1403,
  "month": 10,
  "monitoringPeriod": "ماهانه"
}

فرمت پاسخ دریافتی وب سرویس

ساختار ثابت پاسخ 
{
  "httpCode": 200,
  "message": "SUCCESS",
  "data": {
    "planValue": 90,
    "actualValue": 88.1
  }
}
وقتی مقدار httpCode برابر ۲۰۰ باشد فراخوانی موفق است و همسو مقدار برنامه‌ای و واقعی را از data.planValue و data.actualValue دریافت می‌کند. خطاهای بیزینسی نیز با مقدار متفاوت در httpCode و متن قابل نمایش در message برگردانده می‌شوند.

نگاشت دریافت اتوماتیک مقادیر شاخص

فیلد همسو مسیر در پاسخ سرویس مبدا نوع داده نمونه مقدار وضعیت
مقدار واقعی ۸۷.۵ نگاشت شده
مقدار برنامه‌ای ۹۰ نگاشت شده
پیام وضعیت SUCCESS سیستمی

Authorization سامانه مبدا

در صورت فعال بودن Authorization، همسو ابتدا سرویس دریافت توکن را با user/pass فراخوانی می‌کند و سپس سرویس مقدار شاخص را اجرا می‌کند.
